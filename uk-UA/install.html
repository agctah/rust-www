Встановлення Rust

Для того, щоб встановити Rust, виконайте наступне у вашому терміналі і далі слідуйте інструкціям, що з'являться на екрані.

curl https://sh.rustup.rs -sSf | sh

Для того, щоб встановити Rust, завантажте та виконайте rustup‑init.exe і далі слідуйте інструкціям, що з'являться на екрані.

Не вдалось розпізнати вашу платформу.

Rust виконується на Windows, Linux, Mac OS X, FreeBSD та NetBSD. Якщо ви користуєтесь однією із цих платформ і бачите це повідомлення, будь ласка, повідомте про ваду, разом із наступними значеннями:

navigator.platform:
navigator.appVersion:

Вказівки щодо встановлення Rust

Керування інструментами за допомогою rustup

Rust встановлюється і керується інструментом rustup. Rust має швидкий 6-тижневий процес випуску релізів і підтримує велику кількість платформ , тож багато збірок Rust доступні одночасно. rustup керує цими збірками на кожній платформі, яку підтримує Rust, дозволяючи встановлення Rust із бета та нічних реліз-каналів, до того ж підтримуються додаткові цілі крос-компіляції.

За більш детальною інформацією дивіться rustup документацію.

Конфігурування змінної середовища PATH

В середовищі розробки Rust всі інструменти встановлюються у директорію ~/.cargo/bin %USERPROFILE%\.cargo\bin і це те місце де ви зможете відшукати інструменти розробки Rust, включаючи rustc, cargo і rustup.

Відповідно, розробники Rust, зазвичай, додають цю директорію до їх змінної середовища PATH. В процесі встановлення rustup спробує налаштувати PATH. У зв'язку з відмінностями між платформами, командними оболонками і вадами у rustup, модифікації PATH можуть не дати ефекту доки консоль не буде перезапущено або поточний користувач не завершить сеанс чи навіть взагалі успіху не буде досягнуто.

Якщо після встановлення, виконання rustc --version в консолі завершується невдачею - найбільш ймовірно це з вище наведених причин.

У випадку Windows

У Windows Rust додатково вимагає C++ build tools for Visual Studio 2013 або вище. Найлегший спосіб отримати дані інструменти - це встановити Microsoft Visual C++ Build Tools 2017 , що встановить лише Visual C++ build tools. Як альтернативу ви можете встановити Visual Studio 2017, Visual Studio 2015 або Visual Studio 2013 і під час встановлення обрати "C++ tools".

Для більш детальної інформації щодо налаштування Rust у Windows дивіться специфічну для Windows rustup документацію.

Інші шляхи встановлення

Процес встановлення, описаний вище, за допомогою rustup - це найкращий спосіб встановлення Rust для більшості розробників. Однак Rust також може бути встановлена іншими способами.
